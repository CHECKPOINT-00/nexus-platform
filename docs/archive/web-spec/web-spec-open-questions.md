# Nhật ký quyết định, giả định, câu hỏi mở, và scope phase sau  ## Mâu thuẫn đã được giải quyết  ### 1. Public, private, hay hybrid  Ban đầu có hướng thiên về nội bộ FPT, sau đó chốt thành hybrid.  Kết luận cuối:  - public landing page để ai cũng xem được; - private workspace để xử lý case thật.  ### 2. Không account hay full account  Ban đầu có phương án case-link không cần mật khẩu.  Kết luận cuối:  - dùng Better Auth ngay; - role gồm `guest`, `user`, `supporter`, `admin`; - không bắt buộc email FPT.  ### 3. Form, chat, hay form giả lập chat  Ban đầu có nhiều phương án.  Kết luận cuối:  - dùng conversational structured intake; - action quan trọng phải mở form riêng; - chat tự do chỉ là kênh trao đổi, không phải nguồn dữ liệu chính.  ### 4. AI-only hay human-in-the-loop  Kết luận cuối:  - AI-assisted + supporter-controlled; - supporter duyệt output quan trọng trước khi gửi.  ### 5. n8n là core hay phụ trợ  Kết luận cuối:  - core workflow chạy trong Hono backend; - Vercel AI SDK dùng trong Hono; - n8n chỉ cho automation phụ.  ### 6. Drive hay database là source of truth cho file  Kết luận cuối:  - Drive là source of truth cho file lifecycle; - Postgres mirror metadata và workflow.  ## Giả định đang dùng trong spec này  - `guest` là trạng thái chưa đăng nhập, không phải account riêng. - `auth_user_id` là khóa tham chiếu chính sang Better Auth. - Team là đơn vị trả tiền chính ở MVP. - Một case thường gắn với một nhóm hoặc một đại diện nhóm. - Report phải có supporter review trước khi gửi.  ## Câu hỏi còn mở cần chốt trước khi build xong MVP  ### Pricing cuối  Transcript có nhắc khoảng giá thử nghiệm 100k đến 300k+, nhưng bảng giá cuối chưa thấy chốt.  ### Ranh giới support nội bộ  Cần chốt mức nào của prompt, rubric, và nội dung workflow được phép xuất hiện công khai dưới dạng marketing.  ### File types phase 1  Cần chốt file nào hỗ trợ ngay, file nào để phase sau, nhất là upload trực tiếp so với Drive link.  ### SLA và turnaround  Cần chốt thời gian xử lý kỳ vọng cho từng package, vì đây ảnh hưởng trực tiếp tới định vị và bảng giá.  ### Mức granular của admin console  Admin cần bao nhiêu màn ở MVP, và cái nào chỉ để phase sau.  ## Scope phase sau, chưa vào MVP  - realtime chat kiểu messenger; - n8n automation sâu; - payment gateway tích hợp ngay; - analytics nâng cao; - Google Meet automation; - AI agent tự động toàn phần; - upload file trực tiếp như kênh chính nếu team chưa muốn đi xa phase 1.  ## Chốt đọc nhanh  Nếu team cần khóa spec để build, phần đã đủ chín là:  - hybrid public/private; - Better Auth; - structured conversational intake; - AI-assisted + supporter-controlled; - team/case/package pricing; - Drive lifecycle + Postgres mirror; - report lifecycle; - MVP cutline tối giản.  Phần nên review thêm là:  - giá cuối; - ranh giới public content; - chi tiết file type và SLA; - mức sâu của admin console.
